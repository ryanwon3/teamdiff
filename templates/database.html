Live ingestion health, timeline coverage for gold @15 and curves, and recent matches

Live — summary auto-refreshes every 30s (when this tab is visible).

Newest row in the table is the most recent ingest. Timeline rows must exist for gold leaderboards and curves on the home page.

Gold features (how this DB is used)

  • Lane pairing uses team_position and Riot participant_id on each participant row.
  • Gold at ≤15 min reads participant_timeline.total_gold at the latest stored minute ≤ 15 for each side (same rule as the API).
  • Curves average total_gold (and level for level mode) across games where both laners have a frame at the same minute.

Matches without timeline still count in the DB but cannot power gold stats until the collector fetches timelines for new games. A match can show timeline rows while participant rows still lack lane / Riot id (older ingests); run backfill_participant_meta.py or keep collect_matches.py running so those fields get filled from match detail.
